Purpose

This simulator helps visualize deadlock detection in operating systems using the wait-for graph algorithm. It's designed for educational purposes to understand resource allocation and deadlock scenarios.

Algorithm

Uses cycle detection in wait-for graphs to identify deadlocked processes. When a process Pi waits for resources held by Pj, an edge is created, and cycles indicate deadlocks.

Features

Interactive matrix inputs for allocation and request tables, real-time deadlock detection, visual feedback with cycle identification, and comprehensive process analysis.
